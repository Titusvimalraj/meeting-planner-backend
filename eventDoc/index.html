Events to listen

verifyUser event
This event ("verifyUser") has to be listened on the user's end to verify user authentication.
user will only be set as online user after verification of authentication token.
online-user-list event
This event ("online-user-list") has to be listened on the user's end to identify all available users that are currently online.
All available users can be shown to the user based on which he can start chatting with a user.
The output will be an object, object has key as userId and value as full name.
userId event
This event's name is actually dynamic and refers to the userId of the logged in user.
This event ("userId") has to be listened to identify an individual notification and the update from the admin that has been received.
Example output it as follows:

                                "primary": "#ff5959",

error-occurred event
This event ("error-occurred") has to be listened to identify an error which will occur when the data is not sent properly in the events to be emitted.
Example output it as follows:
                            {
                                status: 123,
                                error: 'error message'
                            }
disconnect event (optional)
This event ("disconnect") has to be listened, when the user is disconnected to show proper message.

Events to emit

set-user event
This event ("set-user") has to be emitted when a user comes online.
User can only be set as online only after verification of authentication token. Which you have pass here. The following data has to be emitted
authentication token
event-add event
This event ("event-add") has to be emitted to add an event to the database.
The following data has to be emitted
                           {
                            "color": {
                                "primary": "#80ff80",
                                "secondary": "#ff5959"
                            },

socket-send-email event
This event ("socket-send-email") has to be emitted, when the admin user does add or edit or delete event.
The following data has to be emitted
                           {
                            email: email of user bruce@example.com,
                            name: Bruce Wayne,
                            subject: `titus vimal raj -Admin has cancelled your meeting: First Event`,
                            html: `<h3> Meeting cancelled </h3>
                            <br> Hi , Bruce Wayne.
                            <br> titus vimal raj -Admin has cancelled your meeting: First Event.
                           `
                          }
socket-reminder-email event
This event ("socket-reminder-email") has to be emitted, when the admin user does add or edit or delete event.
The following data has to be emitted
                           {
                            email: email of user bruce@example.com,
                            name: Bruce Wayne,
                            subject: `titus vimal raj -Admin has sent reminder for your meeting: First Event`,
                            html: `<h3> Meeting Reminder </h3>
                            <br> Hi , Bruce Wayne.
                            <br> titus vimal raj -Admin has sent reminder for your meeting: First Event.
                           `
                          }
notify-updates event
This event ("notify-updates") has to be emitted, when the admin user does add or edit or delete event.
The following data has to be emitted
                            {
                            message: `Hi, titus vimal raj- Admin has cancelled the meeting - First Event.
                             Please Check your Calendar/Email`,
                            userId: this.userId
                          }
